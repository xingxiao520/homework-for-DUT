以下内容是关于开源协议的笔记  
## GPL系
修改完GPL开源协议的代码后必须保持开源   
例：Linux WordPress Git  
传染性：如果一个项目使用了部分GPL代码，整个项目必须遵循GPL协议  
LGPL：宽松版GPL协议，新增部分的代码不必要使用源码协议，使GPL代码不会传染整个项目  
## 其他系
Apache协议：需要为每个修改后的文件放置版权说明  
MIT协议：只需在项目中保留一部分协议的副本就可以几乎随意使用开源代码
BSD协议：在MIT的基础上，不可以使用原作者的名字对项目进行促销推广
